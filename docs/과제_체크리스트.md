# Term Project 과제 체크리스트

과제 공지사항을 기준으로 작성된 최종 체크리스트입니다.

## 1. 기능 범위 및 제약사항 (필수)

### 1-1. 리소스 구성
- [x] **핵심 리소스 4개 이상** 정의 및 구현 (예: User, Post, Comment, Report 등).
- [x] 각 리소스에 대해 **CRUD** (Create, Read, Update, Delete) 기능 제공.
    - [x] Auth (인증/인가 전용)

### 1-2. 엔드포인트 수 (총 30개 이상)
- [x] **총 30개 이상의 HTTP 엔드포인트 구현**.
    - [x] 기본 CRUD (4리소스 * 4 = 16개 이상).
    - [x] 고급 기능 (검색, 정렬, 페이지네이션 포함).
    - [x] 서브 리소스 (예: `/posts/{id}/comments`).
    - [x] 관리자 전용 기능.
    - [x] 집계/통계 기능.
    - [x] 동일 URL이라도 기능이 명확히 다르면 인정 (예: Method 차이).

### 1-3. 인증/인가
- [x] **JWT 기반 인증** 구현.
    - [ ] `POST /auth/login` 구현 X -> OAuth2, Firebase로 구현
    - [x] `POST /auth/reissue` (토큰 재발급).
    - [x] 만료 및 위조 토큰 처리.
- [x] **소셜 로그인 (총 2개 이상)**.
    - [x] Firebase Auth 기반 구현 (최소 1개 이상).
    - [x] 추가 소셜 로그인 1종 (Google, Kakao, Naver 등). (Google OAuth2 구현됨)
- [x] **Role 기반 인가 (RBAC)**.
    - [x] 최소 2개 Role (`ROLE_USER`, `ROLE_ADMIN`, `ROLE_OWNER`).
    - [x] 관리자 전용 엔드포인트 최소 3개 이상.
        - `GET /admin/reports` (신고 목록 조회)
        - `PATCH /admin/reports/{id}` (신고 처리)
        - `GET /admin/statistics` (시스템 통계 조회)
        - `PATCH /admin/users/{id}/role` (사용자 ROLE 변경)

### 1-4. 요청 검증 & 에러 응답 규격
- [x] **입력 검증**: DTO/Schema를 통한 필드 검사.
- [x] **에러 응답 통일** (JSON 포맷).
    - `timestamp`, `path`, `status`, `code`, `message`, `details` 포함.
- [x] **대표 에러 코드 최소 10종 이상** 정의.
    - [x] 400 (`BAD_REQUEST`, `VALIDATION_FAILED`, `INVALID_QUERY_PARAM`)
    - [x] 401 (`UNAUTHORIZED`, `TOKEN_EXPIRED`)
    - [x] 403 (`FORBIDDEN`)
    - [x] 404 (`RESOURCE_NOT_FOUND`, `USER_NOT_FOUND`)
    - [x] 409 (`DUPLICATE_RESOURCE`, `STATE_CONFLICT`)
    - [x] 422 (`UNPROCESSABLE_ENTITY`)
    - [x] 429 (`TOO_MANY_REQUESTS`)
    - [x] 500 (`INTERNAL_SERVER_ERROR`, `DATABASE_ERROR`, `UNKNOWN_ERROR`)

### 1-5. 목록 조회 공통 규격
- [x] **페이지네이션**: `page`, `size` 지원.
- [x] **정렬**: `sort=field,DESC|ASC` 형식 지원.
- [x] **검색/필터**: 최소 2개 이상 조건 지원 (`keyword`, `category`, `date` 등).
- [x] 응답 규격 준수 (`content`, `page`, `size`, `totalElements` 등).

### 1-6. 데이터베이스
- [x] **MySQL** (또는 RDB) 사용.
- [x] **외래키(FK)** 설계.
