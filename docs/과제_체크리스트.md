## 2. 필수 구현 항목 (Implementation)

### 2-1. API 인터페이스 및 규격
- [ ] **엔드포인트 수**: 30개 이상 구현 (단순 CRUD + 검색/정렬/하위리소스 포함)
- [ ] **HTTP Status Code 12종 사용 확인**:
    - [ ] 200 (OK)
    - [ ] 201 (Created)
    - [ ] 204 (No Content)
    - [ ] 400 (Bad Request)
    - [ ] 401 (Unauthorized)
    - [ ] 403 (Forbidden)
    - [ ] 404 (Not Found)
    - [ ] 409 (Conflict)
    - [ ] 422 (Unprocessable Entity)
    - [ ] 429 (Too Many Requests)
    - [ ] 500 (Internal Server Error)
    - [ ] 503 (Service Unavailable)
- [ ] **에러 응답 규격 통일**:
    - [ ] JSON 포맷 준수 (`timestamp`, `path`, `status`, `code`, `message`, `details`)
    - [ ] Validation Error 시 `details`에 필드별 원인 포함
- [ ] **필수 에러 코드 구현**:
    - [ ] BAD_REQUEST
    - [ ] VALIDATION_FAILED
    - [ ] UNAUTHORIZED / TOKEN_EXPIRED
    - [ ] FORBIDDEN
    - [ ] RESOURCE_NOT_FOUND / USER_NOT_FOUND
    - [ ] DUPLICATE_RESOURCE / STATE_CONFLICT
    - [ ] UNPROCESSABLE_ENTITY
    - [ ] TOO_MANY_REQUESTS
    - [ ] INTERNAL_SERVER_ERROR / DATABASE_ERROR

### 2-2. 목록 조회 표준 (List Operations)
- [ ] **페이지네이션**:
    - [ ] Query Param: `?page=...&size=...` 동작
    - [ ] `size` 파라미터 미입력 시 기본값 적용
    - [ ] `size` 최대값 제한 적용 (예: max=100)
    - [ ] 응답에 `page`, `totalElements`, `totalPages` 포함
- [ ] **정렬 (Sorting)**:
    - [ ] Query Param: `?sort=field,desc` 형식 지원
- [ ] **검색 (Filtering)**:
    - [ ] 최소 2개 이상의 검색 조건 (예: `?keyword=...`, `?category=...`)

### 2-3. 인증 및 인가 (Security)
- [ ] **JWT 인증**:
    - [ ] Access Token 발급 (`POST /auth/login`)
    - [ ] Refresh Token 발급 및 사용 (`POST /auth/refresh`)
    - [ ] 토큰 만료 에러 처리 (`TOKEN_EXPIRED`)
- [ ] **소셜 로그인**:
    - [ ] Firebase Auth 연동 구현 (1개 이상)
    - [ ] 일반 OAuth(Google, Kakao 등) 또는 Firebase 추가 구현 (총 2개 이상)
- [ ] **RBAC (Role Based Access Control)**:
    - [ ] `ROLE_USER` 권한 로직
    - [ ] `ROLE_ADMIN` 권한 로직
    - [ ] 관리자 전용 API 최소 3개 (예: 사용자 관리, 통계, 시스템 설정)

### 2-4. 데이터베이스 및 성능
- [ ] **DB 설계**:
    - [ ] 핵심 리소스 4개 이상 (예: User, Product, Order, Review)
    - [ ] 외래키(FK) 설정 완료
    - [ ] 검색/정렬 조건 컬럼에 인덱스(Index) 생성
- [ ] **데이터 처리**:
    - [ ] 시드 데이터 200건 이상 생성완료
    - [ ] N+1 문제 해결 (Batch fetch, Eager loading 등)
- [ ] **Redis 활용**:
    - [ ] 용도 명확화 (캐시, 세션 저장소, Rate Limiting 중 택1 이상)
